## Проблема → решение → ценность

**Проблема:** пользователю нужно самому выбирать модель, обрабатывать голос, файлы и картинки и помнить контекст между сессиями.

**Решение:** GPTHub — единый чат-воркспейс: текст, голос, изображения и файлы в одном потоке; в **v3** запросы идут через **orchestrator** в **LiteLLM** (единый gateway к провайдерам); память/RAG по необходимости.

**Ценность:** пользователь думает о задаче, а не о стеке моделей.

## Что сказать жюри про безопасность (кратко)

> Запросы к LLM идут через **LiteLLM**; опциональный guardrail **prompt injection** в LiteLLM для длинного текста из **RAG/PDF** даёт ложные срабатывания, поэтому в текущем конфиге он **отключён** (см. комментарий в `litellm/config.yaml`). Это осознанный trade-off для стабильного демо с документами; **не замена модерации**. Публичное демо закрываем **ограничением регистрации** (`ENABLE_SIGNUP`, роли), сильными секретами и без публикации внутренних URL в git. Подробнее: [TEAM_PUBLIC_ACCESS.md](TEAM_PUBLIC_ACCESS.md).

## Что сказать жюри про «как работает роутинг»

Используйте формулировку, совпадающую с **v3**:

> Пользователь работает в **Open WebUI**. Дальше запрос идёт в **оркестратор (FastAPI)**: простой классификатор и политика ролей подставляют **имя модели-алиаса** из нашей матрицы. Оркестратор вызывает **LiteLLM Proxy**, где заданы реальные провайдеры (MWS, OpenRouter, локальный instruct и т.д.) и **fallback**. Список моделей в UI приходит с того же пути (`GET /v1/models` через оркестратор). ASR и эмбеддинги для RAG — отдельные OpenAI-совместимые endpoint’ы в настройках WebUI, без второго «мозга» выбора LLM.

## Сценарий демо 60–90 секунд (прогнать заранее)

Цель: одна команда, 2–3 вложения, **без live-таймаутов** (при необходимости кэш промежуточных результатов).

1. Открыть **работающий продукт** по публичной ссылке.
2. Загрузить **короткий аудиофайл** (или заранее вставленный транскрипт + указать, что ASR уже отработал).
3. Загрузить **PDF или изображение** (короткие, тестовые данные без чужих ПД).
4. Одна инструкция пользователя: например *«Суммируй вложения и предложи план действий»*.
5. Показать ответ и блок **trace** (если включён): модель, источники, шаги.

**Запасной план:** записанное демо-видео (для Платформы обязательно для кейса MWS GPT) + на защите короткий live только по стабильной ветке.
